## GIS与BIM融合的研究进展与发展趋势（武鹏飞）

- 核心观点：单独用一个技术满足不了智慧城市的精细管理，GIS 能看全局、BIM 能看建筑细节，结合起来能覆盖从单栋楼到整个城市的不同尺度管理需求。

- 结合方法（3 种）：
  - 转格式：把 BIM 常用的 `IFC` 格式转成 GIS 常用的 `CityGML` 格式（或反过来），但转换时会丢几何信息（如建筑形状细节）和语义信息（如对同一构件的理解不一致）。
  - 扩标准：给 `CityGML` 或 `IFC` 加功能，比如扩展出“城市信息模型（CIM）”“室内应急模型”等，避开转格式丢信息的问题，但仍有语义歧义。
  - 本体论：统一两者对同一对象的理解与定义（例如都把“墙”定义清楚），缓解语义混乱，但该方法还在探索阶段。

- 结合后的主要用途（4 类）：
  - 规划设计：用 GIS 看城市大场景，BIM 评估建筑日照、交通流量，优化设计方案。
  - 工程管理：跟踪建筑材料与构件，管理项目从设计到维护的全流程。
  - 市政管理：管理水管、电缆等市政设施，覆盖不同尺度的建筑/区域。
  - 火灾应急：规划室内外逃生路线、模拟火势蔓延，辅助救援与逃生。

- 现实情况：上述用途多仍在“理论设想”阶段，尚未广泛落地应用。

- 问题与未来方向：
  - 解决融合时的信息丢失（如采用更灵活的数据模型）。
  - 用本体论统一语义，减少语义歧义。
  - 结合物联网（引入传感器实时监测）深化应用。
  - 开发新的数据模型与工具，支撑工程化落地。

## BIM模型到多细节层次GIS模型转换方法（吕慧玲）

- 核心目标：将建筑领域的精细三维 BIM 模型（按 `IFC` 标准）顺畅转换为地理信息领域的 `CityGML` 模型，并覆盖 `LOD1`-`LOD4` 四个精细度层级，解决几何不准、信息丢失与层级覆盖不足等问题。

- CityGML 的 LOD 层级：
  - LOD1：最粗略的棱状块模型，仅用墙面、天花板、楼层信息得到建筑外轮廓与高度；数据量很小（实验完整转换约 13.7KB）。
  - LOD2：在 LOD1 基础上细化屋顶及附属结构，可见柱、梁，外轮廓与屋顶形状更准确。
  - LOD3：进一步细化外观，增加墙体、门、窗，以及屋顶凹陷/突出等细节，可清晰看到门窗构件。
  - LOD4：最精细，加入室内结构（房间、内墙面、天花板、地板、内部门窗、楼梯、家具等），区分内外墙面；因保留大量室内细节，数据量接近原 BIM 模型。

- 转换方法：
  - 明确两种模型的部件差异与各 LOD 层级需求，建立对应关系；
  - 简化 BIM 数据、按 GIS 标准重构几何（如自动区分内外墙）；
  - 完成语义映射，保证构件含义一致。

- 实验与效果：
  - 成功将 `IFC` 模型转换为 `LOD1`-`LOD4` 的 `CityGML` 模型；
  - 几何精度提高、语义对齐、数据量得到优化；
  - 未来工作：补充构件属性（如防火属性），支持室内火灾救援等场景。

## 基于GIS+BIM+IoT数字孪生的堤防工程安全管理平台研究（饶小康）

- 核心思路：融合 `GIS`、`BIM`、`IoT` 构建堤防的数字孪生体，实现精准、实时的安全管理。

- 解决的传统问题：
  - 传统方式以数据库或二维图纸为主，信息不全、精度不足、反馈滞后、可视化有限。

- 技术融合与数据整合：
  - GIS 管宏观地理环境（堤防位置、周边地形等）；
  - BIM 建精细三维模型（内部结构、构件级信息）；
  - IoT 传感器实时采集（水位、降雨、渗流等），与模型和地理数据整合，形成与真实堤防实时对应的数字孪生。

- 险情识别（管涌）模型：
  - 构建深度置信网络（`DBN`）模型，选取 8 个关键影响因素（如水位差、土层厚度）进行判别；
  - 相比 `BP` 神经网络与 `SVM`，精度更高，决定系数 `R²` ≈ 0.912。

- 平台架构（五层）：
  - 物理层：真实堤防与各类传感器；
  - 数据层：数据存储与管理；
  - 业务逻辑层：仿真、分析与决策；
  - 数字孪生体：三维可视化、数据查询、预警展示；
  - 应用层：用户界面（网页、移动端、`VR/AR`）。

- 应用与效果：
  - 在长江干堤典型堤段测试，实时融合地理数据、BIM 模型与传感器数据；
  - 支持三维展示、险情定位、管涌风险评估与预警；
  - 相较传统方式更实时、更精准，辅助管理人员研判与决策，实现智能安全管理。

## 融合BIM与GIS的三维空间数据可视化研究（曹祎楠）

- 问题背景：Web 端同时加载、渲染大规模 `GIS` 数据与大体量 `BIM` 数据困难，提出一套让两者高效融合可视化的方法。

- BIM 与 GIS 的互补与矛盾：
  - 互补：`GIS` 擅长宏观空间管理，`BIM` 提供建筑微观细节与构件属性。
  - 矛盾：`BIM` 数据量大、结构复杂，直接在 Web 端渲染易卡顿；两者 `LOD` 定义不同（GIS 的 LOD 侧重可视化展示，BIM 的 LOD 侧重生命周期阶段），导致融合难。

- 解决方案：
  - 统一 LOD 分层：把 `GIS` 的 LOD 划分思路用在 `BIM` 上，将 `BIM` 拆成 3 层子模型——粗糙外轮廓（`LOD2`）、精细外轮廓（`LOD3`）、室内设施（`LOD4`）；`GIS` 侧用 `ArcGIS` 绘制“矢量白模”（`LOD1`）。形成从宏观（GIS 白模）到微观（BIM 室内）的 4 级模型。
  - 转成三维瓦片：选择 `3D Tiles` 作为两者的“集成容器”，将上述分层模型都转换为 `b3dm` 格式的瓦片（能保留 `BIM` 的属性信息，降低传输压力）。同时设计空间索引，让瓦片按“视距”动态调度：
    - 远距离：仅加载粗糙的 `GIS` 白模（`LOD1`）。
    - 中距离：替换为 `BIM` 外轮廓（`LOD2/LOD3`）。
    - 近距离：靠近单体建筑时才加载其室内设施（`LOD4`）。
  - 开发可视化系统：基于 `Cesium` 引擎的 Web 端系统，支持模型定位、距离测量、路线漫游、构件信息查询等功能。

- 实验验证与性能对比：
  - 渲染时间显著缩短：原模型直接渲染需约 80,000+ 毫秒，`LOD1` 仅约 3,000+ 毫秒。
  - 帧率提升：`LOD1/LOD2` 能稳定在 60 帧，原模型约 20 帧。
  - 保留 `BIM` 的精细属性同时解决 Web 端渲染卡顿问题。

## 基于Cesium的BIM与实景三维模型集成可视化研究（方孟元）

- 文档主题：将建筑模型（`BIM`）与真实环境的实景三维模型（如无人机航测生成的地形/房屋，属 `GIS` 范畴）进行集成，并基于 `Cesium` 在 Web 端清晰展示；以磁浮铁路项目为案例说明具体实现方法。

- 两个核心“零件”：
  - BIM：建筑的“数字孪生”，精细展示结构（柱、梁等），但只关注建筑本体，未覆盖周边环境（如山坡、道路）。
  - 实景三维模型：对真实环境的高精度“复刻”，可视化真实（地形、房屋等），但难以展示建筑内部精细结构。

- 为什么要融合：设计/施工/规划需要同时关注“建筑本体 + 周边环境”。例如磁浮铁路隧道：需既了解隧道口山体现状（实景模型），也掌握隧道内部结构（BIM 模型），两者结合才能判断设计合理性与施工风险。

- 三个关键问题与解决方案：
  1) 格式要统一：`BIM` 与实景模型的文件格式不同，`Cesium` 不一定直接识别；需先将两者都转换为 `Cesium` 可识别的 `3D Tiles`（如 `b3dm` 等），以实现统一加载与展示。
  2) 位置要对齐：`BIM` 常用局部坐标（以建筑某基点为原点），实景模型使用全球坐标（地理坐标系）；直接叠加会错位（如桥“漂浮”在空中）。需将 `BIM` 的局部坐标转换到全球坐标系，与地球坐标精准对齐，使两者正确叠放（桥梁落至实景地面）。
  3) 避免“穿模”：实景模型表示现状（如隧道口是山坡），`BIM` 表示设计方案（如修隧道需开挖山坡）；直接叠加会出现“山坡穿隧道”。需对实景模型中的施工区域进行处理（如“压平”/裁剪），模拟施工后的形态，使 `BIM` 与实景在视觉上自然融合。

- 可视化交互：
  - 动态卷帘（分屏对比）：左侧显示“BIM + 实景融合效果”（如隧道与压平后的山坡），右侧显示“原始实景”（未压平的山坡）；支持拖动分屏线调整比例，对比“设计后”与“现状”，便于评估与方案讨论。

- 总结与价值：
  - 通过统一格式（`3D Tiles`）、坐标对齐与施工区域处理，实现 `BIM` 与实景三维模型在 Web 端的直观融合展示；
  - 支撑建筑设计、路桥施工、城市规划等场景的综合判断与决策；
  - 为构建“智慧城市”平台的多源三维数据融合与可视化奠定技术基础。

## 基于Cesium的三维展示与查询平台开发（赵慧峰）

- 基于 `Cesium` 搭建“网页版三维地图查询系统”，在网页端实现三维场景展示与信息查询；以中国矿业大学校园为示例，将无人机倾斜摄影生成的 3D 模型与多源数据整合到可交互的 Web 平台。

- 背景与动因：
  - 数据获取便捷但展示滞后：无人机倾斜摄影（多角度拍摄快速构建 3D 模型）流行、低成本；传统二维地图不够直观，早期 Web 3D 需插件、加载慢、体验差。
  - 技术条件成熟：`WebGL` 与 `HTML5` 让网页免插件跑 3D；`Cesium` 作为免费框架可搭建“虚拟地球”，叠加 3D 模型与地图数据并交互。
  - 实际需求：学校缺少“三维校园”，外部用户难以在线查看校园 3D 场景并查询建筑信息（学院归属、师资等）。

- 核心工作（四步）：
  1) 数据格式转换（让 Cesium 能识别）：
     - 无人机 3D 模型（如 `osgb`）需转换为 `Cesium` 支持的格式：
       - `glTF`：通用 3D 模型格式，包含形状、颜色、纹理，加载速度快；
       - `3D Tiles`：适用于大场景（如整座校园），将模型切为“瓦片”，利用 `LOD` 远近分级加载，避免网页卡顿；
     - 转换工具：使用 `CesiumLab` 将倾斜摄影模型一键转换为 `3D Tiles`。

  2) 三类查询功能（用户无需翻资料手册）：
     - 几何信息查询：鼠标悬停显示经纬度；点选两点计算直线距离（如从教学楼到图书馆距离）。
     - 属性信息查询：对校园建筑进行“单体化”（可选中、高亮，如变绿）；后端连接 `MySQL` 数据库，存储每栋楼的楼层名称、学科方向、师资人数等；用户点击高亮建筑弹出表格，查看详情。
     - 全景链接查询：采集校园 720° 全景照片（实验室内部、操场等），在 3D 场景对应位置放置“标记点”；点击标记点弹出全景窗口，提供沉浸式实景查看。

  3) 三层系统架构（确保系统可用与扩展）：
     - 客户层（前台）：HTML5 页面；中间展示 3D 校园场景；顶部功能按钮（如“查属性”“换地图底图”）；右侧视角控制（旋转地球、复位）；底部显示经纬度；交互逻辑使用 `JavaScript` 实现（如点击建筑触发查询）。
     - 服务层（后台）：`Node.js` + `Express` 作为中间层；前台发起查询请求，服务层访问数据库，返回结果供展示。
     - 数据存储层（仓库）：静态文件（3D 模型、全景照片、标记点坐标等）与数据库（`MySQL` 存建筑属性信息）。

  4) 外网可访问（突破局域网限制）：
     - 初期仅校园网可访问；为支持校外访问，采用“内网穿透”技术：
       - 工具：`Ngrok`（或国内 `Sunny Ngrok`）；将局域网服务器映射到公网地址；
       - 访问方式：校外用户通过类似 `xxx.ngrok.io` 的地址访问校园内的三维系统，无需学校申请公网 IP。

- 总结与效果：
  - 搭建了可落地的 `Cesium` 三维查询系统：免插件的 Web 访问、3D 校园展示、建筑信息查询、实景全景查看，并支持外网访问；
  - 为校园资产展示、教学导览、访客服务提供直观平台；
  - 后续可优化方向：增加测量工具（面积、角度等）、引入动态模拟（如校园路线导航）。

## 基于Cesium.js和天地图的三维场景构建方法（孙晓鹏）

- 文档主题：使用 `Cesium.js` 结合天地图资源，在网页端构建可观看的三维地形场景，并加载 `3D` 建筑模型；面向 `WebGIS` 的实用流程与方法论。

- 背景与动机：
  - WebGIS 普及，用户希望浏览器中观看立体地形与真实影像叠加；早期依赖插件（如 `Flash`）或浏览器兼容性问题导致体验差。
  - `WebGL` 原生三维渲染成熟，`Cesium.js` 作为基于 `WebGL` 的三维地图库，文档完善、示例丰富，且能与常用开源地图库（如 `OpenLayers`）配合；因此选择其作为三维场景搭建工具。

- 三维场景构建（分三步）：
  1) 影像底图（影像服务）
     - 目标：为三维地球“贴皮肤”。使用天地图资源：
       - 全球影像服务：真实卫星影像，坐标/投影为 `CGCS2000` 与 `Web 墨卡托`，与 `Cesium` 兼容，可直接用作底图。
       - 全球矢量服务：道路、河流、地名标注等，可叠加在卫星底图之上，实现“真实场景 + 标注”的融合显示。
     - `Cesium` 支持多图层叠加：底图为卫星影像，叠加矢量标注提升可读性。

  2) 立体地形（地形服务）
     - 目标：在地图上体现高低起伏（山地/河谷等）。两类途径：
       - 使用现成全球地形：直接调用 `Cesium` 官方高分辨率地形或 `Esri` 全球地形模型（可呈现光照、水面效果）。
       - 自建局部精细地形：针对局部区域使用 `DEM` 数据，处理流程：
         - 用 `Cesium-terrain-builder` 将 `DEM` 转换为 `Cesium` 可识别的地形格式；
         - 用 `Cesium-terrain-server` 发布为地形服务，前端可通过 `Cesium` 加载调用。

  3) 3D 建筑模型（模型加载）
     - 目标：在场景中叠加具体建筑的三维模型（如办公楼）。`Cesium` 支持 `glTF` 模型格式；常用建模软件输出需转换：
       - 将 `3ds Max` 的 `.max` 模型转为 `.obj`，并将纹理改为位图格式（`glTF` 不支持 `tga` 纹理）；
       - 安装 `OpenCollada` 插件，将 `.obj` 转为 `.dae`；
       - 使用 `ColladaToGltf.exe`（`Cesium` 工具）把 `.dae` 转为 `glTF`，即可被 `Cesium` 加载显示。

- 实践效果：
  - 通过天地图的卫星底图 + 矢量标注，在 `Cesium` 中搭建“立体地球”，可直观呈现地形起伏（山地、平原的高低差）。
  - 将“四川省第三测绘工程院”的 `3D` 建筑模型经格式转换加载到场景中，网页端可查看建筑立体外观，并可调整模型的位置与朝向。
  - 全程无插件，支持 `WebGL` 的浏览器（如 `Chrome`、`Firefox`）即可访问，体验便捷。

- 总结与价值：
  - 提供一条面向 `WebGIS` 的现成流程：`调用天地图影像/地形服务 → 转 3D 模型格式 → Cesium 加载显示`；避免从零摸索细节。
  - 对景区三维导览、城市三维地图等项目具备参考价值；可进一步扩展测量（面积、角度）与动态模拟（如路径导航）。

## 基于Cesium的三维GIS网络服务与BIM集成研究（杨李东）

- 文档主题：基于 `Cesium` 搭建一个将“三维 GIS”与“BIM”集成于同一网页的平台，实现城市级宏观场景与单栋建筑微观细节的统一浏览与交互，避免在多个软件间切换。

- 为什么 GIS 与 BIM 要“凑一起”：
  - `GIS`（地理信息系统）：擅长城市级、大范围室外场景的展示与分析（地形、道路、建筑分布），但对建筑内部细节了解有限。
  - `BIM`（建筑信息模型）：提供单体建筑从材料、构件到设备参数的精细信息，但难以与城市级 `GIS` 场景互通。
  - 目标：在一套 Web 平台中既看宏观布局（`GIS`），又能深入到建筑内部（`BIM`），服务智慧城市与建筑运维场景。

- 三个关键问题与解法：
  1) 选对三维引擎（为什么选 `Cesium`）：
     - 免插件：打开浏览器即可用（不依赖 `Flash` 等插件）。
     - 支持 GIS 专业能力：可加载地图与地形，具备距离量测、图层管理等常用操作。
     - 开源免费：可定制以适配 BIM 集成需求。
     - 面向大模型：具备 `3D Tiles` 技术，切片分级加载，保障网页性能。

  2) 解决“语言不通”（BIM → GIS 可识别格式的转换管线）：
     - `IFC`（BIM 常用）→ `CityGML`（GIS 标准，支持多精细层次展示）。
     - `CityGML` → `B3DM`（`Cesium` 专用的 `3D Tiles` 瓦片格式，适合批量加载多栋建筑）。
     - `3D Tiles` 切片分级：将建筑按不同精度切片，先加载粗略版本，再逐步补充细节，提升加载体验并避免卡顿。
     - 效果：在保留 BIM 精细信息（如室内家具、设备）前提下，实现其在 `GIS` 场景中的规范显示与调用。

  3) 搭共用平台（B/S 架构的三层设计）：
     - 数据层：存放地形数据、转换后的 BIM 模型，及基础底图（如天地图、百度地图）。
     - 服务层：提供加载地图、查询 BIM 属性、量测等服务接口；浏览器侧通过请求调用。
     - 表现层：网页界面以 `Cesium` 渲染三维场景；支持在城市 `GIS` 中定位并加载某栋 BIM 模型，进入建筑内部查看细节，执行标绘、属性查询等交互。

- 演示与效果：
  - 首页为 `Cesium` 虚拟地球，可加载天地图与地形，浏览大范围 `GIS` 场景。
  - 加载转换后的 BIM 模型（如两层小楼）：远看为块状，近看呈现窗户等外观细节；可进入查看室内沙发、桌子、柱子等，支持旋转与平移操作。
  - 基础 GIS 功能：距离量测、标绘线条、图层管理（如隐藏地形，仅看建筑）。
  - 性能体验：得益于 `3D Tiles` 切片，即使多模型同时加载，网页也保持流畅。

- 总结与价值：
  - 打通 `GIS` 与 `BIM` 的数据与展示链路，在同一 Web 平台中实现宏观-微观一体化的可视化与查询；
  - 通过标准化转换（`IFC → CityGML → B3DM/3D Tiles`）解决格式不兼容；
  - 免插件、可网络服务化，适用于智慧城市与建筑运维（如消防设施定位与城市位置联动）。
  - 后续优化方向：减少转换信息损失、扩展功能（室内导航、节能分析等），增强工程实用性。
